# IIVCS Glossary

Definitions for the terms, acronyms, and external frameworks referenced across the Insheights Integrated Value Creation System.

---

## Core System and Frameworks

| Term | Definition |
|---|---|
| **IIVCS** | **Insheights Integrated Value Creation System** — the integrated system of organizational and product frameworks (OSM, PTS, CVDM) and measurement models (3E, AER, 8 Values) through which an organization defines, builds, and delivers value. |
| **OSM** | **Organization Strategy Model** — how organizations translate strategic intent into prioritized value creation outcomes, connecting Values, Mission, and Vision through Strategic Value Creation to execution by value stream. |
| **PTS** | **Product Transformation System** — how organizations create value through the interplay of strategy, the product operating model, product management activities, and Product Mindset, enabled by AI. |
| **CVDM** | **Customer Value Delivery Model** — how every customer/user journey step triggers the right value stream stage to deliver the value proposition as a realized customer outcome, with five continuous feedback loops. |
| **POM** | **Product Operating Model** — the operating model PTS stands up: four POM Enablers (Leadership; Practices & Behaviors; Organizational Architecture; Culture & Relationships) that empower persistent cross-functional teams. |
| **Product Mindset** | The operating philosophy of the IIVCS — five principles: customer obsession, outcome over output, continuous discovery, hypothesis-driven thinking, cross-functional ownership. |

## Measurement Models

| Term | Definition |
|---|---|
| **3E** | **Efficiency, Experience, Effectiveness** — the operational value measurement model, sequenced Use → Feel → Achieve, applied consistently across every architecture element. |
| **Efficiency (Use)** | Whether value is accessible without unnecessary friction, effort, cost, or complexity. The table stake; a leading, operational signal. |
| **Experience (Feel)** | Whether value is felt positively — trust, confidence, reduced anxiety. The bridge between Use and Achieve; the earliest leading behavioral signal. |
| **Effectiveness (Achieve)** | Whether intended outcomes are achieved for customer and organization. The proof of value; a lagging, outcome signal. |
| **AER** | **Adoption, Engagement, Retention** — the behavioral measurement model that sits between 3E and the 8 Values, measuring behavioral response to value delivery across two tracks. |
| **Customer AER** | Behavioral response from the customer side — adoption, engagement, retention (or **Recommendation**) of the value offered. |
| **Organizational AER (Org AER)** | Behavioral response from the delivering organization — adoption of the operating model, engagement with purpose and work, retention of talent and institutional knowledge. |
| **8 Values** | **Insheights 8 Organizational Values Measurement Model** — the eight dimensions of Organization Value: Customer, Financial, People, Operational, Market, Strategic, Innovation, Impact. |
| **Feedback System** | The two-level upstream feedback path (operational via CVDM's five loops; organizational via the 8 Values) running CVDM → PTS → OSM. |

## Architecture Elements

The named layers 3E measures and CVDM activates, from broadest to most granular.

| Term | Definition |
|---|---|
| **Value Stream** | The pre-existing business workflows and capabilities through which value is created and delivered — the common organizing spine across OSM, PTS, and CVDM. |
| **Value Stage** | A stage within a value stream, triggered by a customer journey step, that contributes to the customer outcome. |
| **Capability** | An organizational ability that activates within a value stage to contribute to its outcome. |
| **Process** | The repeatable execution that produces a consistent outcome within a capability or stage. |
| **Product / Service** | The delivered offering through which customers access and use value. |
| **Value Proposition** | What a value stream promises to deliver to the customer, realized as a customer outcome. |

## Key Concepts

| Term | Definition |
|---|---|
| **Leading indicator** | A signal that surfaces before outcomes materialize (Efficiency, Experience; AER Adoption, Engagement). Enables intervention before damage registers. |
| **Lagging indicator** | A signal that confirms an outcome after the fact (Effectiveness; AER Retention; most 8 Values signals). |
| **Compounding** | The property that value dimensions reinforce and amplify each other over time, rather than operating independently. |
| **Elements of Value** | Bain & Company's research taxonomy — 30 B2C and 40 B2B elements of what customers value. 3E maps to all 70. |
| **System health indicator** | A signal that confirms a framework is working — generating executable strategy / delivering against priorities / producing customer outcomes — not merely being followed. |

## External Frameworks Referenced

Frameworks the IIVCS comparison documents position against. Listed for reference; see the comparison docs for detail.

| Term | Definition |
|---|---|
| **IR / IIRC 6 Capitals** | International Integrated Reporting Council framework defining six capitals — Financial, Manufactured, Intellectual, Human, Social & Relationship, Natural. The 8 Values bridges to it for external reporting. |
| **Balanced Scorecard (BSC)** | Kaplan & Norton's four-perspective strategic measurement framework (Financial, Customer, Internal Process, Learning & Growth). |
| **ESG / GRI / SASB / TCFD / ISSB** | Environmental, Social, Governance disclosure standards and the bodies that set them (Global Reporting Initiative; Sustainability Accounting Standards Board; Task Force on Climate-related Financial Disclosures; International Sustainability Standards Board). |
| **OKRs** | Objectives & Key Results — a goal-alignment and progress-tracking system. |
| **EFQM / RADAR** | EFQM Excellence Model — European organizational quality framework; RADAR (Results, Approaches, Deployment, Assessment & Refinement) is its scoring method. |
| **TOGAF / BIZBOK / ArchiMate** | Enterprise Architecture frameworks and modeling language. |
| **SAFe** | Scaled Agile Framework — enterprise agile delivery framework. |
| **JTBD** | Jobs To Be Done — a framework for understanding the job a customer hires a product to do. |
| **Service Blueprint / Value Stream Mapping** | Service design and lean process-mapping techniques. |
| **ISO 9241-11** | International usability standard defining Effectiveness, Efficiency, Satisfaction. |
| **Google HEART** | Product UX measurement framework — Happiness, Engagement, Adoption, Retention, Task Success. |
| **AARRR** | "Pirate Metrics" growth funnel — Acquisition, Activation, Retention, Referral, Revenue. |
| **Prosci ADKAR** | Individual change model — Awareness, Desire, Knowledge, Ability, Reinforcement. |
| **Kotter 8-Step** | Kotter's eight-step model for leading organizational change. |
| **McKinsey 7-S** | Organizational alignment model — Strategy, Structure, Systems, Staff, Skills, Style, Shared Values. |
| **Gallup Q12** | A 12-item employee engagement survey instrument. |

## Common Metrics and Acronyms

| Term | Definition |
|---|---|
| **NPS** | Net Promoter Score — stated likelihood to recommend; a proxy for loyalty. |
| **CSAT** | Customer Satisfaction Score — satisfaction with a specific interaction. |
| **CES** | Customer Effort Score — effort required to complete a task; a leading friction signal. |
| **CLV / LTV** | Customer Lifetime Value — total value a customer generates over the relationship. |
| **OEE** | Overall Equipment Effectiveness — a manufacturing performance measure. |
| **GTM** | Go-To-Market — external launch motion (awareness, acquisition, channel activation). |
| **B2B / B2C** | Business-to-business / business-to-consumer. |
| **ERM** | Enterprise Risk Management. |
| **ITIL** | IT service management framework. |
| **IP / R&D** | Intellectual Property / Research & Development. |
