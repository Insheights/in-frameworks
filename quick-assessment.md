# IIVCS Quick Self-Assessment

**A directional reading of your IIVCS adoption maturity in a few minutes — one question per element.**

---

This is the fast self-orientation tool for the [Adoption Sequencing & Maturity Model](adoption-maturity.md). For a rigorous, scored evaluation that produces a full maturity profile, use the [IIVCS Detailed Assessment](detail-assessment.md).

For each IIVCS element, pick the **highest statement that is *fully* true** today. Each maps to a maturity level: **a = Early, b = Developing, c = Mature.** If a statement is only partly true, you have not reached that level — pick the one below. (This mirrors the gated scoring in the [Detailed Assessment](detail-assessment.md): you earn a level by clearing it, not by getting close.)

> **Assess one value stream at a time.** IIVCS maturity is about how the frameworks connect to deliver value — and they connect *on a value stream*. Answer for a single, named value stream, not the whole organization at once.

> **Who answers it.** A single leader can run this quick check for a directional read — but having a few people (strategy, operating model, delivery, measurement) **answer independently and then compare** gives a truer answer: each role sees only its own layer, and where their answers diverge is itself a finding. For the rigorous version of this — independent scoring, evidence, and gated levels with defined roles — use the [Detailed Assessment](detail-assessment.md#who-participates).

---

## The Questions

**1. OSM — Strategy & Governance.** How are your strategic priorities expressed and governed?
- **a)** As initiatives or activities in a plan; not governed by value stream
- **b)** As outcomes mapped to some value streams; reviewed but rarely revised
- **c)** As value-stream-governed outcomes, revised on dual-track AER + 8 Values signal

**2. PTS — Operating Model.** How are delivery teams organized?
- **a)** Project teams by function, assembled and disbanded per initiative, with handoffs
- **b)** Some persistent cross-functional teams on key value streams; partial enabler activation
- **c)** Persistent value-stream teams; all four POM Enablers active; Organizational AER measured

**3. CVDM — Delivery & Realization.** How is delivery organized relative to the customer journey?
- **a)** By internal process; there is no journey-to-value-stream map
- **b)** Key journeys mapped to value streams; some value stages defined; a few loops running
- **c)** Full journey mapping; value stages defined as outcomes; five loops active; Customer AER measured

**4. 3E — Operational Measurement.** How do you measure whether value is delivered at a given element?
- **a)** Activity or output metrics; no consistent measurement language
- **b)** 3E applied at some elements; Use → Feel → Achieve read in places
- **c)** 3E applied consistently across levels and read in sequence diagnostically

**5. AER — Behavioral Measurement.** How do you measure behavioral response — adoption, engagement, retention?
- **a)** Not measured beyond satisfaction surveys
- **b)** One track (usually Customer) measured on some journeys
- **c)** Dual-track (Customer + Organizational) measured in parallel and reconciled at OSM

**6. 8 Values — Organizational Outcomes.** How do you measure the value the organization generates?
- **a)** Financial and operational only, in silos
- **b)** Several dimensions tracked; some traceable to an operational source
- **c)** All 8 Values aggregated from their operational source, feeding OSM governance

**7. Feedback System — Upstream Loops.** What happens to customer and delivery signal?
- **a)** It dies at the team; there is no upstream feedback
- **b)** Operational loops run and reach PTS; signal informs delivery
- **c)** Full upstream path CVDM → PTS → OSM drives strategic reassessment

**8. Product Mindset — Foundation.** What best describes how teams work?
- **a)** Output-focused project culture; assumptions go untested
- **b)** Outcome language adopted; discovery practiced in places
- **c)** Customer obsession, outcome-over-output, continuous discovery, hypothesis-driven thinking, and cross-functional ownership are the norm

## Read Your Result

1. **Count your a / b / c answers.** Mostly **a** → an *Early* system; mostly **b** → *Developing*; mostly **c** → *Mature*. Mixed answers are the most common — and the most useful.
2. **Find your weakest *connected* link.** Not the weakest element in isolation, but the weakest point in the chain strategy → operating model → delivery → feedback (questions 1 → 2 → 3 → 7). That break is where value leaks out of the system, and it is your adoption priority.
3. **Read your starting point** from [Adoption Sequencing](adoption-maturity.md#adoption-sequencing--where-to-start) in the Maturity Model, based on where the break is.

> **Want the full picture?** This quick check is directional only — a one-question read per element, no evidence test. The [IIVCS Detailed Assessment](detail-assessment.md) applies evidence-anchored, gated criteria across all eight components and produces a maturity profile with strengths, constraint, and adoption priority. Use this to orient; use the detailed assessment to decide.

> **What a level means.** Maturity here is **adoption fidelity** — the frameworks present and connected — not proof of value created. A Mature reading means the machinery is in place and wired together, not that the organization is yet creating more value.
